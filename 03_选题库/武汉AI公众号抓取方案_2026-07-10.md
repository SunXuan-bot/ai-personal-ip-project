## 1. 当前结论

这 3 个公众号已收录进 `03_选题库/武汉AI信源表.md`。

抓取上分三层：

| 层级 | 方式 | 稳定性 | 适合做什么 |
| --- | --- | --- | --- |
| 线索发现 | 搜狗微信关键词搜索 | 中等，会受反爬和动态页影响 | 找标题、方向、疑似文章链接 |
| 原文入库 | 用户 / Hermes 转发 `mp.weixin.qq.com` 链接或复制正文 | 高 | 正式整理、摘要、转选题 |
| 事实核验 | 回到官网、公告、活动页、权威媒体 | 高 | 口播前确认事实边界 |

不能假设：只靠 Codex 就能稳定全量爬取公众号历史文章。微信公众号天然有登录态、反爬、跳转链接和版权边界。

## 3. 已做的小测试

### 3.1 搜狗微信账号搜索

命令形态：

```bash
curl -L -s -A 'Mozilla/5.0' 'https://weixin.sogou.com/weixin?type=1&query=创新光谷'
```

结果：

- 非登录态能打开页面。
- 搜索 `创新光谷` 官方认证订阅号时，当前页面显示没有直接命中官方认证订阅号。
- 这不代表公众号不存在，只说明搜狗账号搜索不稳定或认证匹配不足。

### 3.2 搜狗微信文章搜索

命令形态：

```bash
curl -L -s -A 'Mozilla/5.0' 'https://weixin.sogou.com/weixin?type=2&query=创新光谷%20AI'
```

当前能搜到一些相关结果，例如：

- `科技金控·智创未来丨光谷数产发布AI+数据创新生态计划...`
- `赋能AI创新，见证未来启航 | 万企利科创中心成功承办光谷AI盛会`
- `【转载】创新光谷|人工智能一周产业动态`
- `6小时、22支队伍、1个主题：武汉首届AI+游戏大赛在光谷圆满举办!`
- `告别“AI糊涂账”!光谷这家企业推出全国首创DRV...`

说明：

- `创新光谷 AI` 比 `光谷AI一周纵览` 更容易搜到可用线索。
- 搜狗返回的多是 `/link?...` 跳转链接，不一定直接暴露 `mp.weixin.qq.com` 原文。
- 这些结果适合做“线索池”，不适合直接当已核验事实。

## 4. 推荐工作流

### 4.1 每周自动线索搜索

每周跑关键词：

```text
创新光谷 AI
创新光谷 人工智能
光谷AI一周纵览
人工智能一周产业动态
武汉人工智能研究院 AI
中国光谷人才特区 人工智能
```

输出：

```text
03_选题库/公众号线索/YYYY-MM-DD_武汉AI公众号线索.md
03_选题库/公众号线索/YYYY-MM-DD_武汉AI公众号线索.json
```

只记录标题、搜索关键词、搜狗跳转链接、抓取时间、是否疑似来自目标公众号、是否需要用户 / Hermes 补原文。

### 4.2 用户 / Hermes 补原文

当你在微信看到有价值文章时，给 Hermes 的话术：

```text
Hermes，把这篇公众号文章收进 AI个人IP项目的公众号线索：
公众号：
标题：
链接：
我觉得值得看的原因：
如果能复制正文，也一起保存。
```

推荐保存路径：

```text
03_选题库/公众号线索/
```

如果这篇已经明显能做成一期视频，再转成：

```text
03_选题库/点子库/YYYY-MM-DD_短标题/
```

### 4.3 Codex 后续处理

Codex 读取公众号线索后，逐条标记：

| 状态 | 含义 |
| --- | --- |
| `可直接读` | 有 `mp.weixin.qq.com` 链接且 Jina/curl 能读到正文 |
| `需用户补正文` | 链接被微信验证或环境异常挡住 |
| `需官网核验` | 内容重要，但要回政府/光谷/研究院官网确认 |
| `只作选题参考` | 文章有启发，但事实暂不可引用 |
| `放弃` | 来源不清、营销味重、与 AI 主线弱 |

## 5. 半自动抓取脚本

已新增脚本：

```text
00_项目管理/自动化脚本/fetch_wechat_sogou_leads.py
```

定位：

- 输出 JSON / Markdown 供后续人工或 Hermes 补原文。

建议先用于 `创新光谷 AI` 和 `人工智能一周产业动态`。

## 6. 发布前边界

公众号文章可以作为本地线索，但正式视频中要分清：

- 如果来自公众号原文：说“我在某公众号看到一条线索 / 文章提到”。
- 如果已回到官网核验：可以说“某官网 / 某平台发布”。
- 如果只是搜狗标题：只能作为内部线索，不进入口播事实。
