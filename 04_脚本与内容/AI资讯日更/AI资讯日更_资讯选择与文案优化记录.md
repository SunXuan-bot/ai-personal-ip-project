### 文章样本概况

- 标题形态：`OpenAI 发布新语音模型 GPT-Live；粉笔张小龙辞任董事长、CEO；《恋与深空》致歉并宣布退出 BW 展会 | 极客早知道`
- 内容结构：多条科技 / 商业 / 消费电子 / 游戏 / 播客信息集合。
- 头部重点：标题只挑 3 条最有传播性的新闻做组合标题，但正文保留更多条目。
- 单条新闻长度：重点新闻通常 2-4 段，轻量新闻 1 段，信息密度按重要程度分配。
- 来源标注：每条末尾用“来源：某媒体 / 官方 / 公告”形式标注。

### 这篇样本里可见的选题类型

1. AI 模型 / 产品发布：如 OpenAI 语音模型、Grok 编程智能体模型、字节 Seedream、MiniMax 大模型。
2. AI 交互入口变化：语音、智能体、图像生成、AI 眼镜等不是单纯参数新闻，而是代表使用方式变化。
3. 公司与资本动态：如蓝色起源融资、Momenta 港股上市、粉笔高管变动。
4. 消费科技与硬件：如奔驰纯电车型、大疆无人机降落伞、Meta AI 眼镜。
5. 社会争议 / 用户权益：如《恋与深空》道歉、退出展会并承担退票损失。
6. 延伸内容推荐：文章末尾用播客上新承接深度内容，属于资讯流量向长内容转化。

### 先固化的可迁移规则

1. 选题不只看“热度”，还要看是否真的值得目标用户知道。
2. 优质资讯样本应同时满足：来源清楚、主体明确、动作明确、影响对象明确。
3. 新闻正文描述要比普通标题党更扎实：不能只写“某某震撼发布”“行业巨变”，而要讲清楚发生了什么、为什么值得放进今天的 5-6 条里。
4. 对日更口播来说，正文不是越长越好，而是要有足够事实密度，方便压缩成 20-30 秒能听懂的一条。
5. 后续整理资讯时，要把“选择这条的理由”也写出来，便于复盘选题质量，而不只是留下最终口播稿。
6. 标题可以只挑最强 2-3 条做门面，但内部候选池可以更宽；最终口播默认选 5-6 条，质量不足时少讲。
7. 重点条目要写出“为什么重要”：不能只有事件本身，还要补充能力变化、用户规模、价格、开放节奏、业务影响或争议点。
8. 轻量条目可以短写，但必须保留最小事实闭环：时间、主体、动作、结果 / 影响、来源。

### 正文描述方式拆解

这篇样本值得学习的不是文风华丽，而是“新闻事实块”比较完整。适合迁移成我们的候选池写法：

```text
【时间 / 来源】谁发布 / 宣布 / 报道了什么。
【核心事实】产品、功能、参数、价格、融资、职位变化或争议结果是什么。
【补充背景】这件事和行业趋势、公司动作、用户权益或竞争格局有什么关系。
【来源边界】来自官方公告、媒体报道、港交所公告、公司声明等。
```

对 90-150 秒口播的压缩方式：

```text
先用一句话讲事实，再用一句话讲它为什么值得普通 AI 学习者 / 创作者 / 小团队关注。
```

### 对“武汉AI小孙”日更的启发

1. 我们不必照搬“极客早知道”的大而全；它适合做参考池，我们要再筛成更贴近 AI 学习者和实践者的 5-6 条。
2. AI 资讯不要只盯模型名和参数，还要关注交互入口、工具工作流、企业应用、隐私争议和用户权益。
3. 如果一条新闻没有“普通人能感知到的变化”，即使很大，也不一定适合进入当天口播。
4. 口播稿里可以借鉴它的事实密度，但要增加“小孙为什么选它”的解释，形成个人筛选价值。
5. 每条候选新闻都先写完整事实块，再压缩成口播；不要一开始就只写短句，否则容易讲空。

### 对每日候选池文件的新增要求

以后每条候选新闻建议增加一行：

```text

```

写法示例：

```text
选择理由：不是因为标题热，而是因为它代表【模型能力 / 工具工作流 / 国内应用 / 创作者机会】里的一个具体变化。
```

## 2026-07-09 - 用户确认核心胜负手：精准、快速、讲清楚

### 触发点

用户进一步明确：这个栏目要想有播放量、有人关注，核心胜负手还是新闻资讯的精准、快速、讲得清楚，其中“讲清楚”包括语言和画面。

### 我从今天样本中学到的规律

1. 选题要优先命中 AI 圈当天关注的大事，而不是泛科技新闻的平均罗列。
2. 头条必须足够强，能承担标题、封面和开头；其他 4-5 条用于补足信息密度。
3. 优质新闻不是只报“发生了什么”，还要补“为什么重要”：能力变化、开放节奏、用户规模、成本价格、行业影响、争议点。
4. 新闻正文要有事实块：时间、主体、动作、核心事实、背景影响、来源边界。
5. 每条新闻都要考虑画面承载：有官方视频 / Demo 的新闻优先级更高；只有抽象文字、无强关联画面的新闻，除非非常重要，否则不适合作为主新闻。
6. 选择范围要以 AI 为主轴，泛商业、泛消费、泛娱乐信息只有在与 AI 圈判断强相关时才进入。

### 以后筛新闻的硬问题

每条候选先问：

```text
1. 这是 AI 圈今天会关注的大事吗？
2. 它对普通 AI 学习者、创作者、小团队或本地企业有什么关系？
3. 事实能不能快速核验？
4. 我能不能用 10-20 秒讲清楚？
5. 有没有强关联画面素材支撑？
6. 它适合做主新闻，还是只适合做快讯？
```

### 当前边界

我已经从今天样本中学到了“选择与编辑”的第一层规律，但还不能说完全稳定。原因是样本数量仍少，后续需要持续用用户认可 / 不认可的真实稿件校准：

- 哪些 AI 新闻算“大事”。
- 哪些虽然热但不适合本账号。
- 哪种表述最像“小孙筛选”，而不是媒体搬运。
- 哪种画面素材最能帮观众听懂。
