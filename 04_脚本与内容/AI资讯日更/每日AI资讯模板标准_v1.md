# 每日 AI 资讯模板标准 v2

更新日期：2026-07-07  
状态：第二版，用于先把文本结构跑顺  
当前原则：先做新闻密度，不急着做观点密度

## 1. 本模板解决什么

这个模板服务于当前冷启动栏目：

> 每天一分钟，小孙帮你筛几条有用的 AI 信息。

它不是深度解读，也不是观点节目。第一阶段先解决三件事：

- 今天 AI 圈有哪些值得普通人知道的事。
- 哪些事应该放前面，哪些事只适合轻轻带过。
- 口播听起来像新闻播报，而不是像写项目文档。

## 2. 当前结构结论

已调整：

- 每天从候选里筛 5-6 条，不再默认只讲 3 条。
- 不给每条新闻加副标题，也不口播“第一条/第二条”。
- 不逐条写“小孙认为”，避免水词和浅评论。
- 每条新闻稍微展开 1-2 句，讲清楚发生了什么、和谁有关。
- 不在文案里解释为什么某条排第一；排序已经表达重要性。
- 结尾只保留账号名和明天继续，不额外总结；如果最后一条是武汉本地新闻，再接账号定位和交流邀请。
- 语言尽量口语化，保持新闻播报状态。

## 3. 每天最终产物排版

每天新建一个 Markdown 文件，命名：

```text
YYYY-MM-DD_第N天_候选池与口播草稿.md
```

文件建议包含以下模块：

```text
# YYYY-MM-DD 第 N 天：今日 AI 资讯口播稿

生成时间：
状态：
结构原则：

## 今日口播稿

## 今日选题顺序

## 来源与核验备注

## 标题与封面

## 结构调整记录
```

说明：

- 正式给用户读结构时，优先看“今日口播稿”。
- 候选池、素材需求、复盘表可以后置，不要挡在口播前面。
- 等文本结构稳定后，再单独补“素材需求表”和“发布后复盘”。

## 4. 口播结构

目标时长：90-120 秒。

```text
每天一分钟，小孙帮你筛几条今天值得关注的 AI 信息。

【一句话新闻】。
【展开 1-2 句：讲清楚发生了什么，谁发布/谁报道，普通人为什么能听懂】。

【一句话新闻】。
【展开 1-2 句】。

【一句话新闻】。
【展开 1-2 句】。

【一句话新闻】。
【展开 1-2 句】。

【一句话新闻】。
【展开 1-2 句】。

【一句话新闻】。
【展开 1-2 句】。

我是小孙，明天继续帮你看 AI 圈有什么值得关注的新东西。
```

## 5. 单条新闻写法

每条新闻不写副标题，直接进入事实。

推荐：

```text
DeepSeek 官方文档收录了 Deep Code 集成。
Deep Code 是一个面向 DeepSeek-V4 的开源终端 AI 编程助手，GitHub 页面显示，它支持深度思考、推理强度控制和 Agent Skills。
```

不推荐：

```text
AI 编程。
DeepSeek 发布了某某工具。
小孙认为，这说明 AI 正在进入真实工作流。
```

问题：

- “AI 编程”这种副标题没有信息量。
- “第一条/第二条”这种口播标签会显得机械。
- 每条都“小孙认为”会降低新闻密度。
- “真实工作流”这类词容易变成正确但空的水词。

## 6. 选题数量与排序

每天目标 5-6 条。

排序规则：

1. 当天全网热度最高、最可能被目标用户刷到的新闻。
2. 大模型公司、重要工具、AI 编程、AI 视频、AI 办公等高相关信息。
3. 普通人、小团队、内容人能理解或能用上的产品变化。
4. 有争议或有风险的新闻，但要克制表达。
5. 偏技术、偏开发者的信息可以放后面。
6. 宏观行业和就业类信息可以轻点，不要做焦虑标题。

如果热门新闻不在 AI HOT 里，要通过搜索或官方渠道补充，不要只依赖单一入口。

## 7. 结尾标准

默认不做额外总结，不重复讲“今天最关注的是哪条”，也不说“今天先筛这几条”。

原因：

- 新闻排序已经表达了判断。
- 结尾再解释头条，或者再说“今天先筛这几条”，都容易啰嗦。
- 如果前面新闻已经讲清楚，结尾只需要自然收住。

默认结尾：

```text
我是小孙，明天继续帮你看 AI 圈有什么值得关注的新东西。
```

如果当天有武汉本地新闻，且放在最后一条，可以这样收：

```text
如果你也在武汉本地关注 AI，欢迎交流。我是小孙，明天继续帮你筛有用的 AI 信息。
```

## 8. 语言风格

应该像：

```text
今天 AI 圈信息不少，我帮你筛几条重点。
GitHub 页面显示，这个项目目前有 8000 多颗星。
报道称，这次攻击里 AI 参与了入侵、窃取凭证和撰写赎金信等步骤。
这条偏开发者，但能看出 AI 编程工具正在往哪走。
```

避免：

```text
这件事值得普通人关注，不是因为它听起来很热闹，而是因为它可能会进入真实工作流。
这类工具说明一个趋势：AI 不只是聊天窗口，而是在变成具体行业里的工作台。
给普通人的提醒是，先别急着追概念，先看它能不能帮你省时间、提效率、做出结果。
这条今天应该放在最前面，因为它很重要。
```

这些句子可以偶尔用，但不能每条都用。用多了像套话。

## 9. 来源说法

| 来源类型 | 推荐话术 | 避免话术 |
| --- | --- | --- |
| 官方公告 / 官方文档 | “官方文档里出现 / 官方发布 / 官方宣布” | 未确认时不要说“正式发布全新产品” |
| 媒体报道 | “据某某报道” | 不要说成官方宣布 |
| GitHub 项目 | “GitHub 上有一个项目 / 在 GitHub 上受到关注” | 不要夸成行业都在用 |
| X / 社交媒体 | “某某在 X 上提到” | 不要说成确定结论 |
| AI HOT | 作为筛选入口 | 不要把 AI HOT 摘要当原文引用 |

## 10. 发布标题和封面

标题格式：

```text
今天 AI 圈 6 件事：关键词1、关键词2、关键词3
DeepSeek 的 Deep Code 为什么值得放今天头条？
今天 AI 圈信息不少，我帮你筛 6 条重点
```

封面格式：

```text
今天 AI 圈 6 件事
关键词1 / 关键词2 / 关键词3
```

标题优先清楚，不要标题党。

## 11. 发布前检查

- 今天有没有漏掉明显热门新闻？
- 头条是不是当天最该放前面的新闻？
- 新闻数量是否达到 5-6 条？
- 每条是否先讲事实，而不是先讲观点？
- 有没有删掉没必要的副标题？
- 有没有避免每条都评论？
- 有没有删掉“第一条/第二条”这种机械口播标签？
- 有没有避免在文案里解释为什么某条排第一？
- 结尾是不是只保留必要收束；有没有删掉“今天先筛这几条”这类可有可无的话？
- 如果没有本地新闻，有没有避免硬拗武汉定位？
- 语言是不是像人在播新闻，而不是像项目文档？
- 来源边界是否清楚，有没有把二手报道说成官方？

## 12. 对抗式审查

- 这期是不是信息太散，没有主线？
- 有没有为了凑数量放进低价值新闻？
- 有没有漏掉全网正在讨论的大新闻？
- 结尾是不是啰嗦，是否重复评价了头条？
- 是否有生僻词、书面词、水词影响口播？
- 如果观众只听一遍，能不能知道今天 AI 圈发生了什么？
