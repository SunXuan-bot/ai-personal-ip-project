# 周末视频样片试跑指南

日期：2026-07-11  
目标：在家用电脑完成一次“从选题到样片”的真实试跑，验证项目能不能靠当前 GitHub 文档推进，而不是只停留在计划。

## 0. 明天只做一件事

先不要追求完整成片矩阵。

明天的目标是：

```text
选 1 条 AI 资讯主新闻
-> 写 90-150 秒口播
-> 准备最少素材
-> 录制 / 粗剪 / 字幕
-> 做一个可看的视频样片
-> 记录耗时和卡点
```

做到这里，就算试跑成功。

## 1. 开始前先打开这些入口

优先顺序：

1. `00_项目管理/项目动态工作流看板.html`
2. `00_项目管理/AI资讯日更端到端工作流与分工.md`

6. `05_发布与数据/发布资料包/README_发布资料包规范.md`

如果只想少看一点，就先看第 1 个 HTML 和这个指南。

## 2. 明天的推荐流程

### 第一步：确定主新闻

时间盒：30 分钟。

来源可以来自：

- 自动筛选稿。
- AI HOT。
- 你自己看到的公众号 / 视频 / 文章。
- 武汉 AI 信源表里的线索。

只选 1 条主新闻，最多再加 2 条快讯。不要一开始就做 5-6 条完整版。

选择标准：

- 是不是今天 AI 圈值得看的变化。
- 普通 AI 学习者、内容人、小团队能不能听懂。
- 能不能找到官方来源或可信来源。
- 有没有画面素材支撑。

### 第二步：写口播稿

时间盒：45 分钟。

结构：

```text
今天这条 AI 信息，值得你快速看一下。

【谁】发布 / 报道 / 提到【什么事】。
这件事最重要的地方是【一句话解释】。
对普通内容人 / AI 学习者 / 小团队来说，它意味着【一句话关系】。

小孙觉得，【一句判断】。

我是小孙，明天继续帮你筛 AI 里真正值得看的变化。
```

控制在 90-150 秒。

### 第三步：准备最少素材包

时间盒：45-60 分钟。

最低标准：

- 1 个官方页面或可信来源截图。
- 如果有官方视频，下载或记录链接。
- 1 张自制信息卡或标题卡。
- 记录素材来源边界。

素材放到：

```text
02_素材与资料/资讯素材/YYYY-MM-DD_样片/
```

至少写一个：

```text
00_素材台账.md
```

### 第四步：录制 / 粗剪 / 字幕

时间盒：60-90 分钟。

推荐方式：

- 用你熟悉的方式录制真人口播。
- 粗剪掉明显停顿、错读、废话。
- 尽量导出两份：
  - 不带字幕的视频。
  - 独立字幕文件 `.srt`。

如果时间不够，只导出一个带字幕粗剪视频也可以，但要在日报里记录这个限制。

### 第五步：做样片

时间盒：60-120 分钟。

最低样片标准：

- 能看懂这条新闻在讲什么。
- 有真人口播。
- 有字幕或清晰音频。
- 有至少 1 个真实来源画面。
- 有简单标题 / 关键词包装。

不要求明天一定达到正式发布质量。

## 3. 明天必须记录的数据

每一步都记耗时：

| 环节 | 预计耗时 | 实际耗时 | 卡点 |
| --- | ---: | ---: | --- |
| 选题 | 30 分钟 |  |  |
| 核验 | 30 分钟 |  |  |
| 写稿 | 45 分钟 |  |  |
| 素材 | 60 分钟 |  |  |
| 录制粗剪 | 90 分钟 |  |  |
| 包装样片 | 120 分钟 |  |  |
| 发布资料 | 30 分钟 |  |  |

这张表比成片本身还重要，因为第一期目标之一是压缩最小成本时间。

## 4. 明天的完成标准

完成以下 4 件事即可：

- 有一个视频样片文件。
- 有一份口播稿。
- 有一份素材台账。
- 有一条日报，写明今天做到哪、卡在哪里、总耗时多少。

## 5. 明天不要做的事

- 不要一次做完整 30 天规划。
- 不要为了找完美素材拖太久。
- 不要纠结所有平台发布版本。
- 不要在没有核验来源的情况下把二手摘要讲成官方事实。
- 不要把样片失败理解成项目失败；样片就是用来暴露问题的。
