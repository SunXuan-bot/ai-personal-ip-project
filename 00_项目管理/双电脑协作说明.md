# 双电脑协作说明

更新日期：2026-07-17

## 目标

公司本机和家里 Windows 笔记本共同处理本项目时，以 GitHub 私有仓库作为项目进度的同步中枢，避免继续用压缩包交接。

## 权威上下文

项目内 Markdown 是权威上下文，优先级高于任意单台电脑上的 Agent 记忆。

新对话或换电脑继续项目时，先读：

1. `AGENTS.md`
2. `PROJECT_STATE.md`
3. `DECISIONS.md`
4. `DISCUSSION_QUEUE.md`
5. `REFERENCES.md`
6. `RUNBOOK.md`

## 每次开始工作

```bash
git pull
```

然后读取项目状态文件，确认当前目标和下一步。

## 每次结束工作

1. 更新 `PROJECT_STATE.md`：写清楚最近完成、下一步、阻塞。
2. 如有重要选择，更新 `DECISIONS.md`。
3. 将当天产物放入对应文件夹。
4. 提交并推送：

```bash
git status
git add .
git commit -m "更新项目进度"
git push
```

推送后用 `git status` 确认本地分支与 `origin/main` 一致。

## 不进 Git 的内容

以下内容默认不进入 Git：

- 私密人物资料。
- 待整理原始素材。
- 视频、音频和普通图片大文件。
- 压缩包。
- API key、cookie、token、环境变量。
- 归档备份。

如果某个素材必须跨设备同步，优先放云盘，在 Markdown 里记录位置和用途。

受控例外：`05_发布与数据/封面与标题/封面模板/` 中已经确认的小型 PNG / JPG 封面交付资产可以进入 Git，保证另一台电脑能直接查看效果。视频原片和普通中间图片仍不进入 Git。

## 冲突处理

如果两台电脑同时改了同一个 Markdown，先不要强行覆盖。

处理原则：

1. 保留双方新增内容。
2. 以时间更新的 `PROJECT_STATE.md` 为当前进度基准。
3. 无法判断时先问用户，不要自行删除另一台电脑的内容。
