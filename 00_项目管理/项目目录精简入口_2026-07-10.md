# 项目目录精简入口

创建日期：2026-07-10  
目的：把当前偏工程化的文件夹，翻译成用户能快速理解的使用入口。

## 1. 先记住 7 个入口

以后不用先想 `00/01/02/03` 的编号，先按问题找入口：

| 你想做什么 | 去哪里 | 说明 |
| --- | --- | --- |
| 一眼看清项目流程和跑通状态 | `00_项目管理/项目动态工作流看板.html` | 交互式主看板，按工作流从左到右看 |
| 看项目现在做到哪了 | `PROJECT_STATE.md` | 这是总进度，不要到处翻 |
| 想看账号方向和策略 | `01_定位与策略/` | 放大判断、定位、阶段策略 |
| 想丢一个点子 | `03_选题库/点子库/` | 一个点子一个文件夹 |
| 想写一期稿子 | `04_脚本与内容/` | 所有可录制文本都在这里 |
| 想找视频/截图/原始资料 | `02_素材与资料/` | 原始素材和过程素材 |
| 想发布或复盘 | `05_发布与数据/` | 标题、封面、发布包、数据复盘 |

`06_复盘与实验/` 只放实验和技术验证，不作为日常入口。

## 2. 当前目录为什么看起来复杂

现在目录复杂，是因为它同时服务两种需求：

1. 给人看：快速知道方向、选题、脚本、素材、发布。
2. 给 AI / Agent 看：每次接手都能恢复上下文、找到证据、继续工作。

问题是：给 AI 看得越细，用户越容易迷路。

解决办法不是立刻把所有文件都搬走，而是建立一层“人用入口”：

- `PROJECT_STATE.md`：总入口。
- `00_项目管理/项目动态工作流看板.html`：交互式流程总览，用于对齐各工作流进度。
- `00_项目管理/项目目录精简入口_2026-07-10.md`：快速找路。
- `03_选题库/点子库/_汇总与使用记录.md`：所有想法入口。
- `04_脚本与内容/Build_in_Public/`：过程型内容入口。
- `03_选题库/武汉AI信源表.md`：本地信源入口。

## 3. 以后新增内容的落点规则

### 3.1 一个想法

放到：

```text
03_选题库/点子库/YYYY-MM-DD_短标题/
```

里面至少有：

```text
点子卡.md
原始输入.md
使用记录.md
素材/
```

### 3.2 一期要录的视频

放到：

```text
04_脚本与内容/对应栏目/YYYY-MM-DD_标题.md
```

如果只是点子，还没有到可录制脚本，不要放到 `04_脚本与内容/`，先放点子库。

### 3.3 一批原始素材

放到：

```text
02_素材与资料/对应主题/YYYY-MM-DD_短标题/
```

如果是账号搭建过程素材，放：

```text
02_素材与资料/项目过程素材/
```

### 3.4 一个策略判断

放到：

```text
01_定位与策略/
```

策略类文件要少而重，不要每天都新建。日常想法先进点子库。

### 3.5 一次发布

放到：

```text
05_发布与数据/封面与标题/
05_发布与数据/发布资料包/
05_发布与数据/发布复盘/
```

## 4. 建议以后少用的入口

| 位置 | 后续使用方式 |
| --- | --- |
| `06_复盘与实验/` | 只给技术实验、视觉包装、工具测试用。不要把日常策略和选题放进去 |
| `02_素材与资料/待整理/` | 只当临时箱。每周清一次 |
| `04_脚本与内容/节目样片/` | 只保留样片，不再作为日常栏目入口 |
| 根目录零散新文件 | 尽量不要新增。除非是 `PROJECT_STATE.md` 这类长期入口 |

## 5. 每周整理动作

每周只做 15 分钟整理：

1. 看 `03_选题库/点子库/_汇总与使用记录.md`，标记哪些点子已用、待做、废弃。
2. 清一次 `02_素材与资料/待整理/`。
3. 把已经变成脚本的点子，在点子卡里标记 `已转脚本`。
4. 把发布后的数据写进 `05_发布与数据/发布复盘/`。
5. 更新 `PROJECT_STATE.md` 最近完成和下一步。

## 6. 暂不建议立刻搬迁的原因

- 旧文件已经被多个 Markdown 引用。
- 实验目录里有视频、日志、JSON、contact sheet，贸然移动会断引用。
- 先建立“新内容落点规则”，跑 1-2 周后，再决定是否归档旧目录。

如果后续要真正清理，可以按这个顺序：

1. 先清 `待整理/`。
2. 再清过期实验。
3. 最后再处理策略和脚本文件。
