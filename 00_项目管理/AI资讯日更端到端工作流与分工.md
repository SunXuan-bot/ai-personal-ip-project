# AI 资讯日更端到端工作流与分工

更新日期：2026-07-09

目标：把“选题 -> 文案 -> 视频素材 -> 剪辑 -> 封面标题 -> 发布资料包 -> 复盘”整理成稳定流程。当前阶段先支持用户用剪辑软件手工完成成片，Codex 负责前置资料、素材包、发布资料和复盘沉淀。

## 总流程

```text
1. 拉候选
2. 筛 5-6 条
3. 核来源
4. 写口播
5. 做视频素材包
6. 用户录制 / 粗剪 / 字幕
7. 视频包装或剪辑增强
8. 做封面与标题
9. 做发布资料包
10. 发布后复盘
```

## 1. 拉候选

目标：先拿到 10-30 条当天 AI / 科技候选。

Codex 做：

- 从 AI HOT、官方博客、科技媒体、YouTube 官方频道、GitHub、论文页等拉候选。
- 记录来源链接、发布时间、来源类型。
- 标明哪些是官方来源，哪些是媒体二手报道。

用户做：

- 告诉 Codex 当天有没有特别想讲的方向。
- 如果看到好文章、公众号、视频，直接发链接或复制正文。

产物：

```text
04_脚本与内容/AI资讯日更/YYYY-MM-DD_第N期_候选池与口播草稿.md
```

## 2. 筛 5-6 条

目标：默认选 5-6 条，但宁缺毋滥；当天新闻质量不够时，可以少于 5 条。

Codex 做：

- 按“主新闻 / 模型产品 / 实用工具 / 国内动态 / 观点案例 / 备选快讯”初筛。
- 给每条候选补一行“选择理由”。
- 优先保留普通 AI 学习者、创作者、小团队能理解或可能用得上的信息。

用户做：

- 最终拍板当天讲哪 5-6 条，或确认当天只讲 3-4 条。
- 如果不满意，指出是“不够重要”“不够实用”“不好讲”还是“不适合我”。

产物：

```text
最终入选 5-6 条，或当天实际可讲条数
每条选择理由
```

## 3. 核来源

目标：避免把二手摘要、传言或错误信息讲成事实。

Codex 做：

- 尽量找到官方公告、原始视频、官方博客、GitHub、论文页或权威媒体报道。
- 标记核验状态：已核验 / 轻核验 / 仅二手来源 / 待核验。
- 对不确定内容明确写边界。

用户做：

- 对高风险或敏感新闻决定是否放弃。
- 如涉及公司、人物、争议，确认口播措辞是否保守。

产物：

```text
来源核验状态
口播可说边界
```

## 4. 写口播

目标：90-150 秒讲清楚 5-6 条信息；如果坚持 60 秒，则只讲 3-4 条。

Codex 做：

- 先写完整事实块，再压缩成口播。
- 主新闻可稍微展开，其他快讯遵守“事实一句 + 小孙判断一句”。
- 保持“学习者 / 信息筛选者”口吻，不装专家。

用户做：

- 试读，改成自己说得顺的表达。
- 确认是否录制。

产物：

```text
口播稿
标题备选初稿
可录制版文本
```

## 5. 做视频素材包

目标：先把剪辑软件可调用的素材准备好，不急着自动合成。

Codex 做：

- 优先找强关联官方视频素材，例如官方 YouTube、官网 Demo、发布会、产品演示。
- 下载公开视频到本地素材包。
- 找不到视频时，退到官网截图、产品截图、GitHub、论文页、信息卡片。
- 写 `00_素材台账.md`，记录来源、用途、边界和建议片段。

用户做：

- 确认哪些视频素材可以接受用于发布。
- 在剪映 / Premiere / Final Cut 中调用素材。

产物：

```text
02_素材与资料/资讯素材/YYYY-MM-DD_第N期/
```

## 6. 用户录制 / 粗剪 / 字幕

目标：得到一条干净的人声主视频和可用字幕。

Codex 做：

- 根据录制稿协助做字幕校对文本。
- 如果用户提供 `.srt`，可检查术语、英文名、分段。
- 如果需要，可根据字幕辅助拆新闻时间段。

用户做：

- 录制真人口播。
- 粗剪掉明显停顿、错读和废话。
- 在剪映中生成并校对字幕，或导出 `.srt` 给 Codex 检查。

产物：

```text
02_素材与资料/测试视频/或当期视频目录
04_脚本与内容/AI资讯日更/字幕校对稿
```

## 7. 视频包装或剪辑增强

目标：把口播、字幕、素材组合成可发布视频。

Codex 做：

- 可做 HyperFrames / frontend-slides / 信息卡片 / 素材时间轴建议。
- 可裁切官方视频素材片段，生成 `video/clips/`。
- 可提供“哪句话配哪段素材”的剪辑表。

用户做：

- 当前优先在剪辑软件里完成最终剪辑。
- 判断画面节奏、字幕样式、真人出镜比例是否舒服。

产物：

```text
剪辑工程或导出视频
素材调用表
```

## 8. 做封面与标题

目标：把视频包装成平台上能被理解、愿意点击的内容。

Codex 做：

- 生成标题备选：信息型、利益型、观点型、轻悬念型。
- 生成封面文案：主标题、副标题、角标、关键词。
- 根据视频内容建议封面画面：真人截图、产品画面、信息卡、二者组合。

用户做：

- 选择最终标题方向。
- 选择封面风格，必要时在剪映 / PS / Canva 中完成最终图。

产物：

```text
05_发布与数据/封面与标题/YYYY-MM-DD_第N期_封面标题方案.md
```

## 9. 做发布资料包

目标：发布前所有平台文案集中在一个文件里，不临时乱找。

Codex 做：

- 整理最终标题、简介、话题标签、置顶评论、来源备注。
- 按平台准备不同版本：视频号 / 抖音 / B站 / 小红书。
- 写清素材和新闻来源边界。

用户做：

- 最终确认发布平台和发布时间。
- 手动发布，或明确授权后再讨论自动发布。

产物：

```text
05_发布与数据/发布资料包/YYYY-MM-DD_第N期_发布资料包.md
```

## 10. 发布后复盘

目标：用数据和主观感受决定下一条怎么改。

Codex 做：

- 生成复盘表。
- 根据数据和用户反馈总结：选题、标题、封面、前 3 秒、素材、评论钩子。
- 把确定经验写回 `LESSONS_LEARNED.md` 或对应工作流。

用户做：

- 填播放、点赞、评论、收藏、转发、新增关注、私信等数据。
- 补主观感受：哪里顺、哪里累、哪里不想再做。

产物：

```text
05_发布与数据/发布复盘/YYYY-MM-DD_第N期_发布复盘.md
```

## 当前最小可执行版本

下一期先不要全链路自动化，只跑这 5 步：

1. 选 5-6 条资讯，质量不足时少讲。
2. 为最重要的 1 条找并下载强关联官方视频。
3. 写 90-150 秒口播。
4. 做封面标题方案。
5. 做发布资料包。

跑通后再考虑自动裁切素材、自动生成信息卡、自动合成视频。
