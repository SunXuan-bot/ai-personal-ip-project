# Hermes 点子同步方案

创建日期：2026-07-10  
目标：用户在路上想到的点子，可以通过 Hermes 同步到本项目，Codex 后续接手整理、拆选题、写脚本。

## 1. 总原则

- 一个点子一个文件夹。
- 原始想法不要过度清洗，先保留原话。
- 先保存，再判断好坏。
- Codex 后续负责：归类、补来源、拆角度、转脚本、记录使用状态。

## 2. 推荐路径

所有点子放在：

```text
/Users/sunxuan/Desktop/codex工作区/个人自媒体/AI个人IP项目/03_选题库/点子库/
```

单个点子的标准结构：

```text
YYYY-MM-DD_短标题/
  点子卡.md
  原始输入.md
  使用记录.md
  素材/
```

## 3. Hermes 的三种同步方式

### 方式 A：最稳妥，先写收集箱

Hermes 直接把想法追加到：

```text
03_选题库/点子库/_Hermes收集箱.md
```

适合场景：

- 用户在路上语音输入。
- 想法还很散。
- Hermes 不确定应该怎么命名文件夹。

优点：

- 不容易写错路径。
- 不需要立刻拆结构。

缺点：

- Codex 后续需要二次整理。

### 方式 B：Hermes 直接新建点子文件夹

Hermes 如果能控制本机电脑，可以直接创建：

```text
03_选题库/点子库/YYYY-MM-DD_短标题/
```

并写入：

```text
点子卡.md
原始输入.md
使用记录.md
```

适合场景：

- 用户已经明确这是未来一期选题。
- 有链接、截图、公众号正文或抖音链接。
- Hermes 能稳定写本机文件。

### 方式 C：用脚本创建点子文件夹

已准备脚本：

```text
00_项目管理/自动化脚本/create_idea_folder.sh
```

用法：

```bash
cd /Users/sunxuan/Desktop/codex工作区/个人自媒体/AI个人IP项目
./00_项目管理/自动化脚本/create_idea_folder.sh "短标题" "原始想法内容"
```

脚本会自动：

- 创建点子文件夹。
- 写入 `点子卡.md`、`原始输入.md`、`使用记录.md`。
- 生成 `素材/` 子目录。

## 4. 用户给 Hermes 的推荐话术

### 只想先保存

```text
Hermes，把这个想法记到 AI个人IP项目的点子库收集箱：
【这里说想法】
```

### 想直接建一个选题文件夹

```text
Hermes，把这个作为一个新选题，按“一个点子一个文件夹”写到：
/Users/sunxuan/Desktop/codex工作区/个人自媒体/AI个人IP项目/03_选题库/点子库/

标题：
【短标题】

原始想法：
【完整想法】

素材：
【链接/截图/公众号/抖音/备注】
```

### 想让 Codex 后续处理

```text
Hermes，保存后在收集箱里标注：需要 Codex 后续拆成脚本。
```

## 5. Codex 后续处理规则

每次 Codex 接手点子库时：

1. 先读 `_Hermes收集箱.md`。
2. 把新的收集箱条目拆成独立点子文件夹。
3. 更新 `_汇总与使用记录.md`。
4. 判断是否需要补来源、补素材或转正式脚本。
5. 如果转脚本，在点子卡和使用记录里写明脚本路径。

## 6. 社交媒体内容如何处理

公众号、抖音、小红书、视频号这类内容，Hermes 可以先保存：

- 原链接。
- 截图。
- 复制正文。
- 发布时间和账号名。
- 用户一句话评价：为什么觉得值得看。

Codex 后续处理时分三层：

| 层级 | 处理方式 |
| --- | --- |
| 可核验 | 找官网、公告、权威媒体或原始视频补来源 |
| 可参考 | 只学习选题角度、标题、表达方式，不直接复述事实 |
| 不可用 | 来源不清、疑似营销、无法确认事实，保留但不进入口播 |

## 7. 安全边界

- 不要让 Hermes 写入 API key、cookie、身份证、手机号等敏感内容。
- 公众号全文如果涉及版权，只作为内部参考，不要整篇外发。
- 抖音 / 小红书视频素材如果要用于发布，先确认版权和引用边界。
- 私密人物资料不要进入点子库，放 `02_素材与资料/私密参考/`。
