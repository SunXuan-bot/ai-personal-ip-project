# 2026-07 AI 资讯视频包装实验结论

用途：保留 7 月 8 日至 11 日旧 AI 资讯视频实验中可复用的结论。原始帧、样片、克隆仓库、依赖、脚本缓存和失败版本已删除。

## 为什么停止继续做

- 项目已经从“固定 AI 资讯日更”转向三类真实可持续内容实践。
- 连续技术优化能提高包装，但不能解决选题定位、真实供给和受众连接。
- 实验生成约 3GB、近 3 万个文件，维护成本明显高于当前价值。

## 仍然有效的内容结论

1. 资讯视频的核心不是堆 B-roll，而是统一信息层级、真实证据和清楚口播。
2. 官方视频并不总能命中；找到后也必须抽帧、看 contact sheet、筛强相关时间码。
3. 证据优先级：官方实操 / Demo > 官方页面 / GitHub / 论文 > 一手社交帖 > 可信媒体 > 自制信息卡。
4. 页面截图要聚焦与新闻主体相关的区域，不能把整页网页直接塞进画面。
5. 来源卡只能说明边界，不能长期充当主画面。
6. 对外必须区分官方公开视频、官网嵌入 Demo、本地运行、页面录屏和第三方素材。

## 仍然有效的包装结论

- 真人口播是信任主体，包装只服务理解，不应遮挡人脸和字幕。
- 标题卡适合短句，不适合放完整新闻标题。
- HTML / 浏览器渲染信息卡比临时图片脚本更容易控制排版和字幕安全区。
- 静态截图需要局部放大、焦点移动或分层标注，但动效不能制造不存在的事实。
- 字幕必须用真实时间轴并在长句内部继续切分；按文本长度估算容易错位或截断。
- `ffmpeg/ffprobe` 适合转码、抽帧、裁切、合成和最终音视频校验，不适合独自承担全部视觉设计。

## 工具边界

| 工具 | 保留判断 |
| --- | --- |
| HyperFrames | 能做可控包装和时间线，但不自动解决素材选择与审美；只在有明确小样目标时使用。 |
| 剪映 | 仍适合人工粗剪、字幕校对和最后审美调整。 |
| Edge TTS | 可生成中文测试口播并提供句界；最终真人内容不依赖它。 |
| video-use | 有转写、EDL 和时间线自检价值，但中文、竖屏和 API 成本未验证，不进入当前主流程。 |
| ChatCut | 比脚本式工具更接近可编辑剪辑器，但涉及云端上传、账号、credits 和 beta 稳定性，必须先做非敏感小样。 |
| AI-Youtube-Shorts-Generator | 目标偏爆款片段，不等于能准确提取新闻证据，不进入主流程。 |

## 以后重新启用的条件

只有同时满足以下条件才重启复杂包装实验：

1. 已发布至少 3 条内容，确认某种内容值得持续。
2. 明确要解决一个具体问题，例如字幕、标题卡或素材时间码，而不是泛泛“提高质感”。
3. 先做 20-30 秒小样，用户确认后再扩整条。
4. 实验目录不保存依赖仓库、缓存和中间帧；结束后只留结论和必要最终资产。
