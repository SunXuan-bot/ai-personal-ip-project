# 决策记录

只保留仍然有效的决策。被新判断覆盖的旧决策不继续堆在本文件中。

## 2026-07-04 - 长期项目与隐私隔离

- 项目使用 Markdown 作为跨对话、跨电脑的权威上下文。
- 私密人物资料、客户资料和未公开业务只放本地，不进入公开内容和 GitHub。
- 重要策略同时经过第一性原理和对抗式审查，但不能因分析推迟真实实践。

## 2026-07-06 - 阶段账号名

- 阶段账号名使用“武汉AI小孙”。
- 账号不是“AI 专家”或“AI 老师”，而是真实学习、使用、筛选和解释 AI 的内容工作者。
- 主页简介等首条内容确定后再做一次轻量修订。

## 2026-07-17 - 不追求第一次找准最终定位

- 先从真实可持续供给出发，做 9 条小样，再根据反馈反推定位。
- 第一轮三类内容各 3 条：内容 / 自媒体 AI 技能实测、海外 AI 内容大众化转述、Codex / AI Agent 心得。
- 企业服务脱敏案例和 Clawee / 林塬科技内容低频发布，必须遵守客户、公司和未公开信息边界。

## 2026-07-17 - 内容姿态与目标受众

- 当前定位假设是“真实使用 AI 做内容和运营工作的人”。
- 优先服务非技术内容创作者、运营人员、小团队和想理解 AI Agent 的普通用户。
- 不主攻深度代码、模型原理、论文或 AI 快速赚钱人群。

## 2026-07-17 - 封面视觉方向

- 长期视觉采用“真人可信 + 真实截图 / 文件 + 一句判断 + 栏目角标”。
- 三个栏目识别：`小孙实测`、`海外AI转述`、`Agent心得`。
- 不采用纯蓝紫科技风、强赚钱承诺、训练营海报或完全不露脸的工具截图号风格。

## 2026-07-17 - 项目清理规则

- 当前报告覆盖的旧稿直接删除，不再保留多层本地归档。
- 生成型实验只保留结论 Markdown；帧、缓存、依赖仓库、日志、失败版本和重复样片删除。
- GitHub 只同步文档、脚本和小型交付资产；视频原片、私密资料和大体积媒体继续留在本地。
