# 运行手册

## 换电脑或新对话

1. 执行 `git pull`。
2. 依次阅读 `AGENTS.md`、`PROJECT_STATE.md`、`DECISIONS.md`、`DISCUSSION_QUEUE.md`。
3. 从 `00_项目管理/项目看板.md` 找到唯一的进行中任务。
4. 完成阶段性工作后更新 `PROJECT_STATE.md`，再提交并推送。

## 做一条小样

1. 在 `03_选题库/首轮9条小样实验表.md` 选择一条。
2. 明确它服务谁、解决什么问题、真实来源是什么。
3. 产出标题、封面文案、60-90 秒口播结构和一个具体资料钩子。
4. 优先使用真实界面、真实文件、原视频或官方来源；不为了包装额外制造复杂素材。
5. 录制、剪辑并只做一轮必要修改。
6. 发布后在 `05_发布与数据/发布复盘/` 记录平台数据、评论问题、制作耗时和有效连接。

## 三类内容格式

### 内容技能实测

真实问题 -> 使用什么工具 / prompt / skill -> 省掉什么 -> 哪里不靠谱 -> 适合谁 -> 资料钩子。

### 海外 AI 转述

原视频讲什么 -> 对普通内容人真正有用的点 -> 我的判断 -> 原视频和补充资料。

### Agent 心得

原来怎么理解 -> 实际卡在哪里 -> 怎么改指令 / 文件 / 流程 -> 对普通用户的启发 -> 模板或 prompt。

## 封面

- 小红书默认 3:4，抖音默认 9:16。
- 主标题只表达一个判断，优先控制在 12-16 个汉字内。
- 真人、真实截图或真实文件至少出现一种证据，不做空泛 AI 科技背景。
- 栏目角标固定使用 `小孙实测`、`海外AI转述`、`Agent心得`。

## 来源与公开边界

- 企业案例只讲脱敏后的问题、拆解、判断和结果，不分享客户内部技能。
- Clawee / 林塬科技内容发布前确认是否公开，不暴露客户和内部业务。
- 海外内容必须保留原链接和原作者信息，转述要加入自己的解释和判断。
- 不确定的事实标记待核验，不补写、不猜测。

## 外部动作

以下动作必须由用户确认：公开发布、发送消息、付费、上传私密素材、公开客户或公司信息。
