# 章节划分与审查

## 第一性原理

视频时间戳不是装饰条，而是降低观众理解成本的导航。首帧必须回答：

- 这条视频一共有几段？
- 每段大概从什么时候开始？
- 我现在看到哪里了？
- 后面还有多少没讲？

因此，节点时间、章节标题、移动进度和已播放状态缺一不可。

## 从字幕划分章节

优先读 SRT 的内容转折，而不是平均分段。可按以下信号找节点：

- 主持人口播出现“首先、然后、接下来、现在、最后、总结”等转折词。
- 屏幕内容从一个操作切到另一个操作。
- 前一段问题已经解决，下一段开始提出新问题。
- 讲述从背景介绍进入实操、从实操进入效果展示、从效果展示进入复盘。

## 文案压缩

好标题像导航，不像句子：

- “Codex 的安装”优于“把 GitHub 地址交给 Codex 安装”
- “上传照片让AI换肤”优于“我直接把自己的照片给了 Codex 让它换皮肤”
- “追问整体UI统一”优于“发现只是换了头图后继续追问怎么让整体 UI 统一”

## 对抗式审查清单

- 首帧是否能看清所有节点时间？
- 时间点是否和实际内容切换一致？
- 章节标题有没有超出分隔栏？
- 光标经过后，背景、时间、文字是否都发生颜色变化？
- 已播放区域和未播放区域是否一眼能区分？
- 字幕或原画面中的重要顶部 UI 是否被遮挡？
- 输出是否保留音轨？
- 输出文件是否异常小？
- 章节 JSON 是否能被脚本读取，尤其是 PowerShell 生成文件时的 UTF-8 BOM？
- 是否误覆盖原片？
- 是否把调试截图、临时 ASS、过期视频误当最终交付？
