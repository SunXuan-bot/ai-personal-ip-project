# 视频时间戳

## 目标

给已经剪好的视频添加顶部章节时间戳：观众在首帧就能看到视频分成几段、每段从什么时间开始；播放时进度从左到右移动，经过的区域变白，节点时间和章节文案从白字变黑字。

默认成品样式来自 2026-07-19 的横屏实测版本：

- 顶部横向条贴满视频左右两端。
- 初始状态：黑底、白色竖杠、白色节点时间、白色章节标题。
- 播放状态：已播放区域变白；扫过的时间和文字逐步变黑；黄色竖向光标跟随进度移动。
- 节点时间常驻显示，例如 `00:00 / 00:18 / 00:47`。
- 不覆盖原视频，输出新 MP4。

## 默认流程

1. 读取视频信息：用 `ffprobe` 确认宽高、时长、视频流和音频流。
2. 读取字幕：优先读取同目录 `.srt`，按内容自然段划分章节。
3. 生成章节表：每章包含 `start` 和 `title`，第一章必须从 `00:00` 开始。
4. 运行脚本：用 `scripts/render_video_timestamps.py` 生成 ASS 覆盖层并调用 ffmpeg 渲染。
5. 验证输出：检查新视频有视频流、音频流、时长接近原片，并抽首帧、章节切换后、片尾帧确认效果。
6. 保留原片：除非用户明确要求，不删除或覆盖任何原视频。

## 章节文案规则

- 每段标题保持短，优先 6-10 个汉字，最多不超过 12 个汉字。
- 标题只说这段“在干什么”，不要写解释句。
- 章节数优先 5-8 个；太少看不出结构，太多会挤。
- 时间点选“内容真正切换”的位置，不按平均时长机械切。
- 如果字幕存在识别错误，只在时间轴文案里做必要修正，不改原字幕文件。

## 脚本用法

准备一个章节 JSON，例如：

```json
[
  {"start": "00:00", "title": "99元皮肤其实开源"},
  {"start": "00:18", "title": "Codex 的安装"},
  {"start": "00:47", "title": "效果图要自己生图"},
  {"start": "01:10", "title": "重启查看默认皮肤"},
  {"start": "01:58", "title": "上传照片让AI换肤"},
  {"start": "02:46", "title": "追问整体UI统一"},
  {"start": "03:43", "title": "复制提示词继续改"}
]
```

运行：

```powershell
python .agents/skills/video-timestamp/scripts/render_video_timestamps.py `
  --input "D:\path\成品.mp4" `
  --chapters "D:\path\chapters.json" `
  --output "D:\path\成品_带节点时间戳进度轨迹.mp4"
```

常用参数：

- `--bar-height 68`：顶部条高度，横屏 1080p 默认 68。
- `--title-font-size 21`：章节标题字号。
- `--time-font-size 15`：节点时间字号。
- `--crf 20`：H.264 输出质量，数字越小画质越高、文件越大。
- `--preset veryfast`：编码速度。

## 验证标准

必须至少验证：

- 输出文件存在，且不为异常小文件。
- `ffprobe` 能看到一个视频流和一个音频流。
- 宽高、帧率、时长与原视频一致或接近。
- 首帧：所有节点时间和章节标题为黑底白字，全部可见。
- 中段帧：已播放区域白底黑字，未播放区域黑底白字，黄色光标在合理位置。
- 片尾帧：进度接近最右侧，时间和文案仍可读。

## 对抗式审查

每次使用前检查这些风险：

- 字太长：如果某段标题贴近竖杠，先改短文案，不优先缩到很小的字号。
- 章节太多：超过 8 段时，横屏会拥挤；应合并章节或改成两行/竖屏专版。
- 起点不准：章节时间要跟内容转场匹配，不能只按平均分布。
- 时间戳缺失：节点时间必须常驻显示；不能只显示移动光标。
- 颜色反转失败：黑底时字必须白，白底时字必须黑。
- 音轨丢失：ffmpeg 命令必须同时映射视频流和音频流，避免只输出音频壳。
- 非 16:9 视频：脚本会按视频尺寸缩放，但竖屏版尚未深度优化；竖屏发布前必须抽帧检查安全区和文字尺寸。
- 路径问题：Windows 中文路径可能在控制台显示乱码，但只要文件实际生成即可；ffmpeg 滤镜尽量使用脚本生成的本地 ASS 文件名。
- 章节 JSON 编码：PowerShell 可能写出带 BOM 的 UTF-8 文件，脚本已按 `utf-8-sig` 兼容；如果手写 JSON 报错，先检查逗号、引号和编码。
- 不要误删素材：清理旧版本前必须得到用户明确确认。

## 迭代方向

后续可升级：

- 竖屏 1080x1920 专版布局。
- 更精致的字体、透明度、圆角或轻量阴影。
- 当前章节高亮而非只靠光标。
- 自动从 SRT 初步建议章节，再由用户确认。
