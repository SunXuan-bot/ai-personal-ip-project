# AI 资讯视频素材工作流

更新日期：2026-07-09

目标：每期 AI 资讯先独立准备“可被剪辑软件调用”的本地素材包，不急着自动合成进视频。素材包优先解决画面丰富度和新闻强关联问题。

## 核心原则

素材优先级从 2026-07-09 起调整为：

1. 强关联官方视频素材。
2. 官方页面 / 产品页面 / GitHub / 论文 / 公告截图。
3. 自制信息卡片、数据卡、流程图。
4. AI 生图或抽象视觉补充。

判断标准：

- 能用新闻主体自己的公开视频，就不要只用网页截图。
- 能用官方发布会、官方 Demo、官方频道视频，就不要优先使用二手混剪。
- 找不到强关联视频时，再退到官网截图、产品页截图和信息卡片。
- AI 生图只能表达抽象概念，不能伪造成真实产品、真实人物、真实新闻现场。

## 适用场景

例如新闻是：

```text
OpenAI 发布最新语音模型。
```

优先找：

- OpenAI 官方 YouTube 频道是否发布了相关视频。
- OpenAI 官网 / blog / launch 页面是否嵌入公开视频。
- OpenAI 官方 X / LinkedIn / 新闻稿是否指向视频。
- 相关产品 Demo、发布会片段、官方短视频。

如果能找到强关联官方视频，就下载原视频或可用清晰度视频，保存为本地素材，供剪映、Premiere、Final Cut 或后续 HyperFrames 调用。

## 本地目录结构

每期新建一个素材包：

```text
02_素材与资料/资讯素材/YYYY-MM-DD_第N期/
  00_素材台账.md
  01_新闻短名/
    video/
      source/
      clips/
    screenshots/
    cards/
    notes.md
  02_新闻短名/
    video/
      source/
      clips/
    screenshots/
    cards/
    notes.md
```

说明：

- `video/source/`：下载的原始公开视频素材。
- `video/clips/`：后续如需截取片段，放裁切后的片段。
- `screenshots/`：官网、产品页、GitHub、公告、论文等截图。
- `cards/`：自制信息卡片或数据卡。
- `notes.md`：单条新闻素材说明。
- `00_素材台账.md`：整期素材总表。

## 下载方式

### 方式 A：本机 yt-dlp

适合：公开视频、快速下载、直接存到本机项目目录。

常用命令：

```bash
yt-dlp -f "bv*[height<=1080]+ba/b[height<=1080]/best[height<=1080]" \
  --merge-output-format mp4 \
  -o "AI个人IP项目/02_素材与资料/资讯素材/YYYY-MM-DD_第N期/01_新闻短名/video/source/%(title).120s [%(id)s].%(ext)s" \
  "YOUTUBE_URL"
```

先查元数据：

```bash
yt-dlp --dump-json "YOUTUBE_URL"
```

只搜索候选视频：

```bash
yt-dlp --dump-json "ytsearch5:OpenAI latest voice model official"
```

### 方式 B：KrillinAI / Content Studio 下载

适合：需要走中心机、沿用 KrillinAI 下载链路、后续可能接横屏/竖屏渲染任务。

规则：

- 默认通过 `content-studio-youtube-download` 技能创建 YouTube 下载任务。
- 下载结果以中心机输出目录里的 `origin_video` 为准。
- 不要虚构清晰度参数；当前技能默认按中心机配置下载，目标清晰度为 1080p，但工具 schema 不一定支持强制指定。

### 推荐选择

- 只为了本机剪辑调用：优先本机 `yt-dlp`，直接放进 `02_素材与资料/资讯素材/`。
- 后续要进入 KrillinAI 内容生产链路：优先 Content Studio 下载，记录 `task_id` 和 `origin_video` 路径。

## 候选工具：AI-Youtube-Shorts-Generator

仓库：

```text
https://github.com/SamurAIGPT/AI-Youtube-Shorts-Generator
```

当前状态：

- 仅作为候选工具记录，尚未安装，尚未在本项目验证。
- 主要价值不是帮我们“找新闻素材”，而是当已经找到一条官方长视频后，辅助从长视频中识别并导出精准片段。

可能适用：

- OpenAI / Google / Anthropic / Meta 等官方发布会、访谈、Demo 很长。
- 我们只需要其中 10-60 秒强关联 B-roll。
- 不想先全量下载、手工拖时间轴、再手工裁剪。

可尝试用法：

```text
官方长视频链接
-> 工具转写并识别候选高价值片段
-> 人工确认哪些片段和当前新闻强相关
-> 导出 16:9 横屏素材或 9:16 竖屏素材
-> 存入当期素材包 video/clips/
```

注意：

- 它默认宣传为 YouTube Shorts / 9:16 工具，但支持 `--aspect-ratio`，理论上可导出 16:9。
- 它的选段逻辑偏“爆款短视频”，不等于资讯准确；必须人工确认片段是否和新闻强相关。
- 本地模式依赖 `yt-dlp`、`faster-whisper`、`ffmpeg`、`opencv` 和 LLM API key，依赖较重。
- 使用前要先跑真实小样，再决定是否封装为项目技能。

待验证问题：

1. 是否能稳定导出 16:9 横屏片段。
2. 是否能跳过竖屏裁切，只按原视频比例裁剪。
3. 对官方发布会 / Demo 的片段识别是否比手工找时间码更省时间。
4. 输出 JSON 是否方便写入素材台账。

## 每条新闻的视频素材流程

1. 明确新闻主体和关键词。
2. 优先搜索官方频道或官网嵌入视频。
3. 判断是否强关联：标题、发布时间、发布方、视频内容是否对应这条新闻。
4. 如果视频较短，直接下载原始视频到 `video/source/`。
5. 如果视频较长，优先记录为“片段摘取候选”，后续可测试 AI-Youtube-Shorts-Generator 或用 `ffmpeg` 按时间码裁剪。
6. 写入 `notes.md` 和整期 `00_素材台账.md`。
7. 如用户要剪辑，提供建议片段时间码；如果需要，我再裁切到 `video/clips/`。
8. 如果找不到强关联视频，记录“未找到”，再退到截图和信息卡片。

## 使用边界

- 不下载或绕过付费、私域、登录限制、DRM 或明确禁止下载的内容。
- 官方公开视频也不等于自动可商用；发布前需要按平台规则和账号风险判断。
- 优先使用短片段、评论性/说明性引用，避免整段搬运。
- 视频素材包用于剪辑参考和 B-roll 准备；最终发布版本由用户确认使用边界。

## 和日更工作流的衔接

每日 AI 资讯流程新增一个步骤：

```text
选定 5-6 条新闻，或当天实际可讲条数
-> 逐条核来源
-> 逐条找强关联视频素材
-> 下载 / 截图 / 制卡并写素材台账
-> 用户在剪辑软件中调用
-> 后续再决定是否自动合成
```

最低交付标准：

- 每期至少为最重要的 1 条新闻找强关联官方视频素材。
- 找不到视频时，必须说明找过哪些渠道、为什么没有采用。
- 每个下载视频都要有来源链接和用途说明。
